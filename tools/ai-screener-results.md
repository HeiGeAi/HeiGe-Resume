# AI 筛选测试结果（AI Screener Results）

- 运行日期：2026-06-02
- 方法：6 个互不可见的筛选器子 agent，3 个只看普通版纯文本、3 个只看锻造版纯文本，配同一份 JD（高级后端·支付交易方向）+ 同一份评分 rubric 盲打分。详见 `ai-screener-eval.md`。
- 模型：当次会话主模型（Claude Opus 4.8 级别），独立隔离上下文。
- 输入：`inputs/plain-before.txt` / `inputs/plain-after.txt` / `inputs/jd.txt`。

## 筛选器理由摘录

普通版：
- 沾到支付回调/对账/618 等关键词但全是泛泛参与，缺幂等/分库分表/Kafka/QPS 等硬核证据，无量化成果，停留在初中级。
- 缺幂等/分库分表/Kafka/Spring Cloud/QPS 等硬核关键词，技能全是「了解/熟悉」且零量化成果，达不到高级支付岗。
- 全是泛泛描述，缺核心硬指标和 QPS 量级，技能只写「熟悉/了解」，达不到高级工程师水准。

锻造版：
- 支付交易方向高度对口，幂等/分库分表/对账/6 万 QPS/Spring Cloud 全覆盖且量化扎实，仅四年经验在「高级」职级略偏轻。
- 硬关键词几乎全命中且有量化实证，仅四年经验对高级岗略偏短、Redis 与 Kafka 缺项目佐证。
- 几乎覆盖全部硬性关键词且都有量化战绩支撑，仅清结算和容量规划稍弱，无作弊迹象。

## 读法

锻造版在 JD 匹配（38→93.3）、量化结果（17→91）、清晰度（36.7→92.3）三处拉开最大，正是对着岗位改写加上履历升级该赢的地方。近期性两版接近（日期相同），属预期。筛选器对锻造版也指出了真实短板（四年经验对高级岗偏轻），说明它在认真读、不是橡皮图章。

AI 筛选器日间会波动，分数每次重跑会有出入，但锻造版均值显著高于普通版的方向稳定。复现方法见 `ai-screener-eval.md`。
