# HeiGe-Resume（简历锻造系统）

简历不是一张履历表，是一份只为一个岗位写的提案。

大多数 AI 做出来的简历，是把经历堆成一张流水账：求职意向、教育背景、工作经历、负责什么、参与什么，一栏一栏填满。HR 扫一眼，6 秒，扫不到你最强的那条，简历就进了废纸堆。再加上那个烂大街的双栏模板，跟另外八百份一模一样。

HeiGe-Resume 换一种做法：**把同一段经历升级成既被机器排到前面、又让人记得住的简历。** 一句话说清你是谁，每条经历写你做成了什么，对着岗位改，排成机器读得顺的单栏。

这套方法论就干一件事：**让你升级后的同一条经历，机器和人都买账。**

---

## 招牌内核：机器先看，人再看

做简历之前，先认清 2026 年的现实：**机器在人之前就把你筛过一遍了。**

你的简历先躺在简历库里，HR 和猎头拿关键词搜，搜不到你就等于不存在。等你主动投一个岗位，公司的 AI 先读你的简历、按这个岗位给你打分排序，排在后面人根本翻不到你。约 83% 的公司都这么干。过了机器这关，HR 才花六到八秒扫一眼，扫不到你最强那条，简历就进了废纸堆。

填表思维问「我有哪些经历要列上去」。提案思维问「这个岗位要什么、机器会怎么判我、人六秒会停在哪」。**锻造的功夫，是让升级后的同一条经历，被搜到、被读懂、最后被人记住。**

机器那关也分两种处境，按需要做两份简历：一份泛用版扔进简历库，关键词铺到位，等人搜；一份特化版对着具体岗位改，少而精。两种处境要的东西是反的，一份顾不了两头，打法见 `references/ai-screener-optimization.md`。

六道工序，是这套思维的具体落地。每份简历都要过一遍。

### 1. 一句话定位（The One-Liner）

简历顶部那一句话，钉死你是谁、能解决什么，同时镜像目标岗位的语言。机器靠它判匹配，人靠它决定往下看。

- 公式：什么角色 + 几年 + 最硬的一条标签。例：「五年后端，扛过日活千万的支付系统，最近在做 AI 工程化」。
- 针对岗位调这句话。投架构岗和投业务岗，强调的不该是同一条。
- 它是你的钩子，不是你的户口本。

### 2. 履历升级（The Upgrade Engine）

每一条经历，把「负责什么」改写成「做成了什么」。机器看结果、人也看结果，这一条两边都吃。

- 用 XYZ 公式：**做成了 X，量化成 Y，靠的是 Z**。例：「重构支付链路，成功率从 98.1% 提到 99.6%，一年少赔 200 万」。
- 换担当动词（主导 / 重构 / 搭建 / 操盘），匹配你的真实层级，别硬抬。
- 没有数字也能正当量化（区间 / 频率 / 范围 / 折算），每个数字标「约」且能解释来源。
- 红线：强化措辞可以，编造事实不行。每条经得起面试追问。完整打法见 `references/content-upgrade-engine.md`。

### 3. 对岗定制（Tune to the JD）

为目标岗位改简历，回复率大约能翻到原来的六倍，是回报最高的一步。

- 把岗位要求里反复出现的关键词挑出来，用它的原话复述你的经历。它写「微服务」你就写「微服务」，别只写「后端」。
- 把最贴合这个岗位的经历挪到最前，跟岗位没关系的删掉，哪怕本身挺亮眼。
- 岗位匹配在 AI 打分里占约四成，这一步机器算你更对口、人一眼也看出对口。详见 `references/ai-screener-optimization.md`。

### 4. 机器可读（Machine-Readable）

让 AI 筛选器把你解析对、读得懂、排得高。

- 单栏布局、真文字（可选中可复制）、阅读顺序逻辑正确，不嵌图、不用图标存关键信息。
- 标准小标题（工作经历 / 项目 / 技能 / 教育），近期经历往前放（机器对近 2 到 3 年加权）。
- 别用技能进度条 / 五角星，机器读不出还稀释信号。
- 结构硬规格见 `references/resume-production-spec.md` 的「机器可读」一节。

### 5. 6 秒扫描（The 6-Second Scan）

过了机器进到人眼，让最强信号落在视线第一落点。

- 最强的那条经历、那个数字、那家公司，放在顶部和左上。
- 层级靠字号、字重、留白拉开，让眼睛 6 秒内抓到三个重点：你是谁、最强一条、能不能用。
- 别让所有条目一样重。什么都加粗等于什么都没加粗。

### 6. 一页纪律 + 反作弊体检（One-Page & Anti-Gaming Check）

一页讲完（资深最多两页，第一页必须独立成立）。出货前强制过反模板 + 反作弊 + 诚信体检。

命中任何一条都要改：
- 烂大街的双栏模板，跟八百份一样
- 通篇「负责 / 参与」，没有一个结果和数字
- 没有定位句，或定位是正确的废话
- 技能进度条 / 五角星 / 关键信息嵌图（机器读不出）
- **白字塞关键词、prompt 注入糊弄 AI 筛选器（会被检测并惩罚，本系统拒绝执行）**
- 任意一条战功面试追问答不上来（夸大无据）

完整清单见 `references/anti-template-checklist.md`。**这一步不是可选的，是交付门槛。**

## 使用流程

### 第一步：定调（动手前必做）

写任何代码之前，先和用户对齐三件事：

**问题 1：投什么岗位、什么阶段、有没有 JD？要哪份？**
- 目标岗位 / 行业？这决定哪条经历往前放、定位句怎么写。
- 要泛用版还是特化版？泛用版扔进简历库等人搜，关键词铺到位；特化版对着一个具体岗位改，少而精。有具体 JD 就贴进来做特化版，没有就先做泛用版（打法见 `ai-screener-optimization.md`）。
- 职业阶段：应届 / 3-5 年 / 资深 / 高管？阶段不同，重心完全不同（应届重潜力项目，资深重战功影响力）。

**问题 2：手上有哪些料？**
- 让用户给出：经历、项目、数字、技能、教育。料不够就追问关键数字（项目多大体量、带来什么结果）。
- 没有数字的经历，按 `content-upgrade-engine.md` 的正当量化手法一起把「负责什么」挖成「做成了什么」。

**问题 3：气质方向 + 硬约束？**
- 气质从 `references/aesthetic-directions.md` 选（克制专业 / 工业硬核 / 高定优雅 / 设计感张扬 / 温暖简洁）。多数岗位偏克制专业，设计/创意岗可以张扬。
- 硬约束：一页还是两页？要不要照片？中英文？必须包含的内容？

如果用户说「你定」，就按岗位拍一个稳妥又不平庸的方向，动手前一句话告诉他。

### 第二步：搭骨架 + 升级履历

- 先写那句一句话定位，反复打磨到能钉死。
- 从所有经历里挑出最能打的三五条战功，每条按 `content-upgrade-engine.md` 的 XYZ 公式锻造（做成 X / 量化成 Y / 靠 Z），换担当动词、补可验证的数字。
- 诚信红线不碰：强化措辞可以，编造事实不行，每条经得起追问。

### 第三步：对岗定制（或铺关键词）+ 机器可读

在正经找工作的人，建议两份都出：一份泛用版挂进各家简历库等人搜，每投一个岗位再从特化版改一版。两份用同一批真实经历，差别在选材和详略：

| | 泛用版 | 特化版 |
|---|---|---|
| 一句话定位 | 覆盖面广，带几个常被搜的方向词 | 镜像这个岗位，只留最对口的 |
| 经历 | 经历列全，措辞带上更多可搜的词 | 只留跟岗位最相关的几条 |
| 技能 | 铺满，把真会的技术和同义叫法都覆盖（分库分表也写 ShardingSphere） | 只留这个 JD 要的那几样，不堆 |

- 特化版就对岗定制：把岗位关键词挑出来、用它的原话复述经历、最相关的挪到最前（见 `ai-screener-optimization.md`）。
- 泛用版就铺关键词：技能和方向铺全，常搜的词在几个地方都带上，行业里同义的叫法都覆盖，只覆盖你真会的。
- 两份都要机器可读：单栏真文字、阅读顺序对、标准小标题、近期靠前，不嵌图不进度条。
- 两份的实际样子见 `examples/generalist-resume.html`（泛用版）和 `examples/tailored-resume.html`（特化版），同一个人，一眼看出差别。

### 第四步：写生产级代码

按 `references/resume-production-spec.md` 的硬规格做，要点：
- **PDF 优先**：A4 尺寸，`@page` 精确控制，Cmd+P 导出的 PDF 就是最终投递版，一页就是一页。
- **机器可读**：所有文字是真文字、可选中、可复制，不嵌图、不用图标存关键信息，阅读顺序逻辑正确。
- **中文字体兜底**（PingFang SC / Microsoft YaHei），换电脑、被墙都不崩。
- 字号阶梯严格，对齐到网格，留白克制。

### 第五步：机器关和人关都体检 + 反作弊复检

出货前强制自检：
1. 过一遍 `references/anti-template-checklist.md`，命中的全部干掉。
2. 机器这关：岗位关键词用上了吗、资历和体量写出来了吗、近期经历靠前吗、单栏真文字吗。人这关：六秒能扫到三个重点吗、每条有结果有数字吗、导出 PDF 一页干净吗。
3. 反作弊：没有白字塞词、没有夹带指令糊弄 AI，每条经历面试追问都答得上来。
4. 任何一项不过关，回去改，别投。

## 设计资产（references/）

- `content-upgrade-engine.md`：履历升级。XYZ 公式、强动词三级阶梯、没有数字也能量化的五种正当手法、升级 vs 造假红线。
- `ai-screener-optimization.md`：过机器这关。被搜到（BM25）和被读懂（LLM 语义筛）两种处境、泛用版和特化版两份简历、岗位定制打法、同源 AI 加分点、注入为什么讲透但不做。
- `resume-production-spec.md`：单文件 HTML 简历的硬规格（A4 / PDF 导出 / 机器可读 ATS+LLM / 中文字体兜底 / 字号阶梯）。
- `content-templates.md`：内容速查卡。一句话定位公式、结果不写职责对照表、各阶段战功写法。
- `aesthetic-directions.md`：气质方向库。克制专业 / 工业硬核 / 高定优雅 / 设计感张扬 / 温暖简洁，每种给出色彩 / 字体 / 排版打法。
- `anti-template-checklist.md`：反模板 + 反作弊 + 诚信体检清单。出货前逐条过。

## 成品参考（examples/）

同一套方法论，跨职业、跨阶段、跨气质能产出完全不同的简历。以下样例均为虚构人物，用本方法论现场锻造，可直接打开、Cmd+P 导出 PDF：

- `comparison.html`：普通版 vs 锻造版前后对比。同一段经历升级后，3 个独立 AI 筛选器盲打分更高，附逐条标注为什么机器和人都更买账。
- `generalist-resume.html` + `tailored-resume.html`：同一个人（程屿）的泛用版和特化版两份。泛用版关键词铺满进简历库被搜，特化版对着岗位少而精，一眼看出两份的取舍差别。
- `heige-resume.html`：资深后端工程师。工业 / 硬核气质。
- `pm-resume.html`：产品经理。克制 / 专业气质。
- `designer-resume.html`：设计师。设计感 / 张扬气质。
- `newgrad-resume.html`：应届毕业生。温暖 / 简洁气质。
- `executive-resume.html`：技术高管。高定 / 极简气质。

## 本地验证（tools/）
