# Changelog

本项目所有重要变更记录于此。

## [2.2.0] - 2026-06-04

把「产两份」从文档落到真文件。

### 新增样例
- `examples/generalist-resume.html`：泛用版。同一个人程屿，关键词铺满（技能和方向覆盖全，分库分表也写 ShardingSphere），用来挂进各家简历库被猎头搜到。
- `examples/tailored-resume.html`：特化版。同一个人，只留最对口的几条、镜像目标岗位的词、少而精，用来对着一个具体岗位投。
- 两份用同一批真实经历，差别在选材和详略，一眼看出取舍。都一页、文字可选中、A4 导出 PDF。屏幕上有版本标签（泛用版 / 特化版），标签不进打印的 PDF。

### 流程
- SKILL.md 第三步把「正经找工作就两份都出」写进流程，给出泛用版 vs 特化版在定位句、经历、技能上各怎么取舍的对照表。

## [2.1.0] - 2026-06-03

补上「被搜到」这一层，把文案改成人话。

### 新增能力
- 简历库检索用的是 BM25 算法，关键词在整篇里的密度决定排序。这版补上这一层，给出泛用版和特化版两份简历的策略：泛用版扔进简历库等人搜，关键词铺到位；特化版对着具体岗位改，少而精。两种处境要的东西是反的，一份顾不了两头。
- `ai-screener-optimization.md` 重写：分清「被搜到（BM25）」和「被读懂（LLM 语义筛）」两种机器处境，加上同源 AI 加分点（用同一个模型改的简历，AI 裁判给分更高）。
- 注入（白字 / 零宽字符）这件事：讲清楚它是什么、为什么大概率没用还会被拉黑，产品不做这个功能。

### 文案
- 全项目文案按人话重写一遍。把「双筛」「两道筛」「对岗调频」「对人赢对 AI 赢」这类压缩出来的机器腔，换成人会说的话（要过两道筛选、对着岗位改、机器排得上人记得住）。

## [2.0.0] - 2026-06-02

机器关升级。2026 年投简历，机器在人之前先把你筛一遍：AI 先按岗位打分排序，过了人再花六秒扫。这版让升级后的同一段经历，机器和人两关都过得去。

### 核心
- 招牌内核从「6 秒决定去留」升级为「机器先看，人再看」。
- 五道工序扩成六道：一句话定位 / 履历升级 / 对岗定制 / 机器可读 / 6 秒扫描 / 一页纪律 + 反作弊体检。
- 使用流程加入贴 JD、升级履历、对着岗位改这几步。

### 新增设计资产
- `references/content-upgrade-engine.md`：履历升级。XYZ 公式、强动词三级阶梯、没有数字也能量化的五种正当手法、升级 vs 造假红线。
- `references/ai-screener-optimization.md`：过机器这关。LLM 语义筛权重（岗位匹配 40% / 资历 25% / 量化 20% / 近期 15%）、对着岗位改的打法、明确拒绝白字塞词等作弊。

### 新增对比实证与测试
- `examples/comparison.html`：普通版 vs 锻造版前后对比，双栏并排 + 逐条拆解 + AI 评分面板，自身也 ATS-safe、可打印。
- `tools/`：本地 AI 筛选测试 harness（方法 + JD + rubric + 盲测输入 + 结果），无需外部 API key。3 个互不可见筛选器对纯文本盲打分，同一段经历升级后加权均分 36.2 → 88.9（+52.7），3 个筛选器方向一致。

### 强化
- `resume-production-spec.md`「ATS 友好」升级为「机器可读（ATS + LLM 语义筛）」，结构铁律不变，更严。
- `content-templates.md` 改为速查卡，深度打法下沉到两个引擎文档，新增对岗版定位句。
- `anti-template-checklist.md` 加机器关与诚信体检，终极一问改成机器和人两关都问。

### 诚信
- 强化措辞可以，编造事实不行。白字关键词 / prompt 注入会被机器抓出并惩罚，本系统拒绝执行并劝阻。每条战功都要经得起面试和背调追问。

### 未改动
- 五套样例本身已达标（XYZ + 强动词 + 单栏真文字 + 近期靠前），本版不改其内容，作为方法论的金样保留。

## [1.0.0] - 2026-06-01

首个版本。

### 核心
- 原创方法论「6 秒决定去留」，五道锻造工序：一句话定位 / 写结果不写职责 / 6 秒扫描 / 一页纪律 / 反模板体检。
- 四步使用流程：定调 → 搭骨架（定位句 + 选战功）→ 写生产级代码 → 反模板体检。

### 设计资产
- `references/aesthetic-directions.md`：5 种气质方向库（克制专业 / 工业硬核 / 高定优雅 / 设计感张扬 / 温暖简洁），每种给出色彩 / 字体 / 排版打法。
- `references/resume-production-spec.md`：单文件 HTML 简历硬规格。PDF 优先（A4 / Cmd+P 完美导出）+ ATS 友好（文字可选中 / 不嵌图 / 不用进度条 / 阅读顺序正确）+ 中文字体兜底，生产铁律从第一天内置。
- `references/content-templates.md`：内容模板。一句话定位公式、结果不写职责改写公式、各职业各阶段战功写法。
- `references/anti-template-checklist.md`：反模板体检清单，出货前逐条过。

### 样例画廊（同一套方法论 × 五种职业气质，全部虚构人物保护隐私）
- `examples/heige-resume.html`：资深后端工程师。工业 / 硬核。1 页。
- `examples/pm-resume.html`：产品经理。克制 / 专业。1 页。
- `examples/designer-resume.html`：设计师。设计感 / 张扬。1 页。
- `examples/newgrad-resume.html`：应届毕业生。温暖 / 简洁。1 页。
- `examples/executive-resume.html`：技术高管。高定 / 极简。2 页。

全部为零依赖单文件，A4 排版，文字可选中（ATS 友好），可 Cmd+P 完美导出 PDF，中文带系统字体兜底。每份均经真实 PDF 导出验证页数与文字可提取性。
