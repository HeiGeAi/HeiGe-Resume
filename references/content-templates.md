# 内容模板（Resume Content Templates）

简历的气场一半在排版，一半在文案。这份给出可直接套用的写法公式。
核心原则：**一句话定位钉死你，每条经历写战功不写职责。**

---

## 一、一句话定位公式

放在姓名下方第一行，决定 HR 要不要往下看。

**公式：角色 + 年限 + 最硬的一条标签（按岗位调）**

- 弱：求职意向：后端开发工程师
- 强：五年后端 · 扛过日活千万的支付系统 · 最近在做 AI 工程化
- 弱：有多年产品经验，熟悉互联网产品
- 强：八年 B 端产品 · 从 0 带过两条百万 ARR 产品线 · 擅长把模糊需求拆成能交付的东西
- 弱：应届毕业生，计算机专业，求一份开发工作
- 强：计算机应届 · 一个人写过一个两万用户的小工具 · 想找能快速上手真实项目的团队

要点：一句话里塞进你最能打的那个事实 + 一个数字 / 体量。针对不同岗位换强调点。

---

## 二、结果不写职责 改写公式

每条经历一行，写你做成了什么，不写你负责什么。

**公式：做了什么动作 → 带来什么结果 → 多大体量 / 多少数字**

| 职责写法（弱） | 战功写法（强） |
|---|---|
| 负责支付系统的开发与维护 | 重构支付核心链路，成功率从 98.1% 提到 99.6%，一年少赔约 200 万 |
| 参与用户增长项目 | 主导拉新策略改版，月新增从 3 万做到 11 万，获客成本降 40% |
| 负责团队管理 | 从 0 组建 8 人后端团队，半年把发布周期从两周压到两天 |
| 协助完成数据看板搭建 | 独立搭起全公司经营看板，让管理层决策不再等周报，被三个部门复用 |
| 负责公众号运营 | 接手后半年把公众号从 2 千粉做到 5 万，单篇最高 10 万+ |

要点：
- 能量化就量化（百分比、金额、倍数、人数、时长）。
- 量化不了，就给可验证的结果（被谁复用、上线了什么、拿了什么结果）。
- 动词开头，用「重构 / 主导 / 搭建 / 从 0 做」这种有担当的动词，少用「负责 / 参与 / 协助」。
- 一条经历挑最能打的 2-4 条战功，不是把所有事都列上。

## 三、各阶段重心参考

**应届 / 经验少**：重心在项目、竞赛、实习、潜力。
- 没有大厂经历不丢人，把一个真实做过的项目讲透（做了什么、用了什么、什么结果）比列十门课强。
- 教育、获奖、开源 / 作品可以靠前。定位句强调潜力和上手速度。

**3-5 年 / 骨干**：重心在战功和成长。
- 主线放最能打的两段经历，每段 2-4 条带数字的结果。
- 体现「越做越大」的成长曲线（带的人变多、扛的体量变大、负责的范围变宽）。

**资深 / 专家**：重心在影响力和深度。
- 不再罗列做过什么，而是讲解决过什么级别的难题、做过什么关键决策、影响了多大盘子。
- 可以有「核心成就」单独一栏，把三五条最硬的拎出来。

**高管 / 管理者**：重心在结果和带队。
- 讲业务结果（营收、规模、市场）、组织结果（带多大团队、搭了什么体系）、战略结果（押对什么方向）。
- 用数字说话，一页纸要有分量感，留白要够。

---

## 四、技能怎么写

- 用文字分级或直接列，**不要进度条 / 五角星**（信息量为零，过不了 ATS）。
- 分组：编程语言 / 框架 / 工具 / 软技能，别一锅烩。
- 只列真会的、和岗位相关的。列一堆「了解」反而稀释。
- 例：`后端：Go（精通）· Java · Python` 比 五个进度条强。

---

## 五、常见雷区

- 顶部写「求职意向 / 个人简介」却是一段正确的废话。
- 通篇「负责 / 参与 / 协助」，没有一个数字。
- 把不相关的经历都堆上，稀释了最强信号。
- 技能进度条、性格五角星、兴趣爱好占大半页。
- 邮箱 / 电话做成图标图片，机器读不出。
- 一句话定位写成口号（「追求卓越，永不止步」），不是事实。
